# Project Bootstrap

## When to Use

- 新しい PJ をこのテンプレートから立ち上げるとき
- 方向性、技術、未確定事項を整理してから実装に入りたいとき

## Steps

1. `docs/BOOTSTRAP.md` を開いて最低限の定義項目を埋める
2. `docs/PROJECT.md` に Goal、Scope、Users を反映する
3. `docs/TECH-STACK.md` に仮の技術候補と検証入口を書く
4. `docs/DELIVERY.md` で AI の完了条件と停止条件を確認する
5. 未確定事項は `docs/tracking/common/QUESTIONS.md` に登録する
6. 実装開始条件を満たしたら `docs/TASKS.md` を更新する

## Checklist

- [ ] Goal が一文で説明できる
- [ ] Scope が最低限定義されている
- [ ] 技術候補と検証入口が書かれている
- [ ] 未確定事項が追跡可能になっている
- [ ] AI の停止条件が明示されている
