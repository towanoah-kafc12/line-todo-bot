# RICH MENU DESIGN

## Purpose

このドキュメントは、LINE Bot のリッチメニュー画像で守る見た目ルールを固定するためのものだ。
実装途中で見た目がぶれないことと、操作導線を迷わせないことを優先する。

## Current Rules

### 1. Section Name Is Primary

- `買うもの` と `やること` は上段の主役として大きく表示する
- section 選択カードは 2 枚構成で、左右に並べる
- section 名はカード内で中央ぞろえにする
- 補助説明より section 名の視認性を優先する

理由:
このメニューの役割は「次にどの section を触るか」を即決させることだからだ。

### 2. Small Helper Text Is Removed

- `セクションを選ぶ`
- `全セクションを表示`
- `通常メニューへ戻る`
- その他の小さな補助文

上のような補助テキストは載せない。

理由:
視線が分散して、押すべき場所より説明文のほうが目立ってしまうからだ。

### 3. Text Is Centered

- 見出しは中央ぞろえ
- section 名は中央ぞろえ
- 下段のボタン文言も中央ぞろえ

理由:
左右のカードと下段ボタンの重心を揃えて、画面全体を安定して見せるためだ。

### 4. Operation Color Is Shared Across Sections

- `買うもの` と `やること` で同じ操作は同じ色にする
- 色の意味は `一覧` `追加` `編集` `完了` の種類に寄せる
- section の区別は左右のカラムと見出しで行う

理由:
色で section と操作の両方を表現すると意味が競合するので、操作種別だけに使った方が一目で分かるからだ。

### 5. Dark Mode Palette

- 背景は白を避け、ダーク寄りのベースカラーを使う
- 上段カードは暗い背景の上で判別できるよう、色相差をつける
- ボタンの青はアプリ全体のアクセントとして残す

理由:
白背景はのっぺり見えやすく、section カードの存在感が弱くなるからだ。

### 6. Visual Hierarchy

- 1番目: 画面タイトル
- 2番目: section 見出し
- 3番目: 操作ボタン群

この順で目に入ることを正解とする。

理由:
操作フローが `操作選択 -> section 選択 -> 補助操作` だからだ。

### 7. Main Menu Is Split By Section

- 通常メニューは左右 2 カラムで `買うもの` と `やること` を分ける
- 各カラムに `一覧を見る` `追加する` `編集する` `完了する` を縦積みする
- `削除する` は通常メニューから外し、編集 flow 内で処理する
- 各カラムの操作ボタンは同じサイズ、同じ並び順で揃える

理由:
main menu の時点で対象 section を固定した方が、操作回数も迷いも減るからだ。

## Fixed Labels

現時点ではリッチメニュー画像内の section 名は動的にしない。
固定ラベルは次の 2 つを使う。

- `買うもの`
- `やること`

理由:
このリポジトリの運用では、可変性よりも視認性とメンテの簡単さを優先するためだ。

## Non-Goals

- 画像内で可変 section 名を自動反映すること
- 小さな説明文を増やして学習コストを下げようとすること
- 情報量を増やして 1 画面で全部説明すること
